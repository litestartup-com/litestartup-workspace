Imagine you are launching a new feature.

The skill follows the blog rules, creates Markdown with frontmatter, commits, pushes, and syncs the content to LiteStartup.

Then ask:

## Built for Startup Email, Not Enterprise Bloat

Startup teams need speed, but email still has to be handled carefully. LiteStartup Skill keeps a few important rules in place:

- Email is previewed before sending
- Direct sends require explicit confirmation
- Notification emails work without a custom sending domain
- Custom domain email is used only when you specify a `from` address
- Email content is not committed to git by default because sending is an API action, not a website publish

That design keeps content publishing and email sending connected without pretending they are the same thing. Blogs, docs, website pages, and changelogs belong in git. Sent emails are actions that should be reviewed and confirmed.
